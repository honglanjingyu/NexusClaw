### 测试步骤

1. **启动服务**
```bash
python run_api.py --port 8002
```

2. **打开另一个终端，发送测试请求**

```bash
# 测试简单问题
curl -X POST http://localhost:8002/api/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "你好", "stream": false}'

# 测试历史查询
curl -X POST http://localhost:8002/api/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "我都问过你什么问题了", "stream": false}'

# 测试知识检索
curl -X POST http://localhost:8002/api/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "什么是人工智能", "stream": false}'

# 测试复杂问题
curl -X POST http://localhost:8002/api/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "帮我分析一下这个月CPU使用率变化趋势，找出异常点", "stream": false}'
```

3. **观察日志输出**

查看控制台日志，应该能看到类似输出：

```
# 简单问题
IntentRouter 初始化完成: use_llm=True
规则匹配: simple (置信度=0.95)
简单意图 (simple)，使用快速响应模式

# 历史查询
规则匹配: history (置信度=0.85)
简单意图 (history)，使用快速响应模式

# 知识检索
规则匹配: knowledge (置信度=0.70)
知识检索意图，使用单次检索模式

# 复杂问题
规则匹配: 未匹配到，使用 LLM 判断
复杂意图，使用完整规划模式
```

4. **测试流式响应（观察首字时间）**

```bash
# 流式测试 - 简单问题（应该很快看到首字）
curl -N -X POST http://localhost:8002/api/v1/chat/stream \
  -H "Content-Type: application/json" \
  -d '{"message": "你好"}'

# 流式测试 - 复杂问题（应该先看到意图事件）
curl -N -X POST http://localhost:8002/api/v1/chat/stream \
  -H "Content-Type: application/json" \
  -d '{"message": "帮我规划一下今天的任务"}'
```

### 期望的响应时间对比

| 问题类型 | 优化前 | 优化后 |
|---------|--------|--------|
| "你好" | 2-3秒 | <500ms |
| "我都问过你什么问题了" | 30-40秒（可能有超时） | <1秒 |
| "什么是人工智能" | 5-10秒 | 1-2秒 |
| 复杂规划问题 | 10-20秒 | 10-20秒（不变） |

### 修改规则的快速测试

如果某个问题分类不符合预期，直接修改 `app/brain/intent_rules.py`：

```python
# 例如：添加新的关键词
IntentType.SIMPLE: [
    "你好", "您好", ...,
    "嗨", "哈喽",  # 新增
],

IntentType.HISTORY: [
    ...,
    "对话记录", "聊天记录",  # 新增
],
```

然后重启服务即可生效。
